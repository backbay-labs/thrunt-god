  <action>
Finish the install flow in `bin/install.js` using the staged bundle from Plan 01:

1. Add helper `getObsidianPluginDir(vaultPath)` returning:
   - `path.join(vaultPath, '.obsidian', 'plugins', OBSIDIAN_PLUGIN_ID)`
2. Add helper `linkObsidianBundleIntoVault(vaultPath, stageDir)` with concrete behavior:
   - ensure `getObsidianPluginDir(vaultPath)` exists (`fs.mkdirSync(..., { recursive: true })`)
   - for each asset in `OBSIDIAN_ASSET_FILES`, compute:
     - source = `path.join(stageDir, asset)`
     - target = `path.join(pluginDir, asset)`
   - if `target` is already a symlink whose realpath resolves to `source`, leave it in place and record `skip` for that asset
   - if `target` exists and is a broken symlink, wrong symlink, or regular file, remove it before relinking and record `success` after replacement
   - create a symlink from target -> source using `fs.symlinkSync(source, target)`
   - verify `fs.lstatSync(target).isSymbolicLink()` after creation
   - return a structured per-vault result that can distinguish `success`, `skip`, and `failure`
3. Update `installObsidian()` so it:
   - stages the bundle first via Plan 01 helpers
   - calls `discoverObsidianVaults()`
   - if the returned vault list is empty, prints:
     - no vaults detected from `obsidian.json`
     - manual-install fallback using `VaultFolder/.obsidian/plugins/thrunt-god/`
     - returns without creating any vault plugin directories
   - otherwise iterates vaults, links the bundle into each vault, and tracks per-vault success/skip/failure
   - prints one line per vault showing `installed`, `skipped`, or `failed` status and the vault path
   - ends successful runs with: `Restart Obsidian and enable THRUNT God in Community Plugins.`
4. Under `THRUNT_TEST_MODE`, export `getObsidianPluginDir`, `linkObsidianBundleIntoVault`, and `installObsidian`.
  </action>

  </verify>
  <acceptance_criteria>
    - `getObsidianPluginDir(vaultPath)` targets `.obsidian/plugins/thrunt-god`
    - `linkObsidianBundleIntoVault` leaves already-correct symlinks alone as `skip` outcomes and replaces stale targets as `success` outcomes
    - `installObsidian()` stages the bundle before reading vault metadata
    - no-vault fallback prints manual-install guidance and does not create vault plugin directories
    - output includes explicit per-vault `installed`, `skipped`, or `failed` result lines
    - successful output includes the exact restart/enable handoff message
