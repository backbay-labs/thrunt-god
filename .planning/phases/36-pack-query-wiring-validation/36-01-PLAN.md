**1. QUERY_STARTERS constant** -- Object keyed by connector ID (all 10), each value is `{ language, template, description }`:

```javascript
const QUERY_STARTERS = {
  splunk:       { language: 'spl',            template: 'index=<index> tenant={{tenant}} earliest=-{{lookback_hours}}h\n| <your_search_logic>\n| stats count by <fields>', description: 'SPL search with tenant and time filter' },
  elastic:      { language: 'esql',           template: 'FROM <index-pattern> | WHERE tenant == "{{tenant}}" AND @timestamp >= NOW() - {{lookback_hours}}h', description: 'ES|QL query with tenant and time filter' },
  sentinel:     { language: 'kql',            template: '<Table>\n| where TimeGenerated > ago({{lookback_hours}}h)\n| where <filter>', description: 'KQL query against Log Analytics' },
  opensearch:   { language: 'sql',            template: 'SELECT * FROM <index> WHERE tenant = \'{{tenant}}\' AND @timestamp >= NOW() - INTERVAL {{lookback_hours}} HOUR', description: 'SQL query with tenant and time filter' },
  defender_xdr: { language: 'kql',            template: '<Table>\n| where Timestamp > ago({{lookback_hours}}h)\n| where <filter>', description: 'Advanced Hunting KQL query' },
  crowdstrike:  { language: 'fql',            template: 'event_simpleName=<EventType> aid:* <filter>', description: 'Falcon Query Language filter' },
  okta:         { language: 'api',            template: 'eventType eq "<event_type>" and <filter>', description: 'Okta System Log API filter' },
  m365:         { language: 'odata',          template: '/<endpoint>?$filter=<field> eq \'{{value}}\'', description: 'Microsoft Graph API OData filter' },
  aws:          { language: 'api',            template: 'eventSource=* userIdentity.arn=*{{focus_principal}}*', description: 'AWS CloudTrail event filter' },
  gcp:          { language: 'logging-filter', template: 'protoPayload.authenticationInfo.principalEmail:{{focus_principal}}', description: 'Cloud Logging filter expression' },
};
```

**2. ENTITY_SCOPE_TYPES constant** -- Array of objects `{ kind, source, description }`:
- 13 runtime extraction kinds (source: 'runtime'): user, host, ip, device, cloud-account, azure-resource, cloud-resource, gcp-resource, principal, resource, alert, file, artifact
- 7 proposed scope types (source: 'proposed'): process, session, sender, domain, mailbox, url, geo
- Each entry has a `description` field (one-line description of the entity type)

**3. getQueryStarter(connectorId) function** -- Returns the QUERY_STARTERS entry for the given connector ID, or `null` if unknown.

**4. runIncrementalValidation(partialPack, checkpoint, options) function**:
- `checkpoint` is one of: 'identity', 'attack', 'query', 'final'
- Calls `packLib.createPackDefinition(partialPack, { allowPartial: true })` to normalize
- Calls `packLib.validatePackDefinition(normalized, { requireComplete: checkpoint === 'final' })`
- For 'query' checkpoint: also calls `packLib.getPackTemplateUsage(normalized)` to check for undeclared template parameters
- Returns `{ checkpoint, passed: boolean, results: Array<{ status: 'PASS'|'WARN'|'FAIL', message: string }> }`
- Maps validation errors to FAIL, warnings to WARN
- Adds checkpoint-specific PASS messages:
  - 'identity': "Pack ID format valid" (if no id error), "Kind-specific requirements met" (if no kind error)
  - 'attack': "ATT&CK technique IDs valid" (if no attack errors)
  - 'query': "All template parameters declared" (if undeclared.length === 0), "Execution targets reference valid connectors" (if no connector errors)
  - 'final': "Full schema validation passed" (if valid)

**5. formatValidationResults(validationResult) function**:
- Takes the output of `runIncrementalValidation()`
- Returns a formatted string with `[PASS]`, `[WARN]`, `[FAIL]` markers per result line
- Format: `"  Validation (checkpoint):\n  [STATUS] message\n  ...\n  Result: VALID/INVALID (N warnings)"`

**Module exports:** `{ QUERY_STARTERS, ENTITY_SCOPE_TYPES, getQueryStarter, runIncrementalValidation, formatValidationResults }`

Import `packLib` from `./pack.cjs` at the top. Do NOT import `runtime.cjs` -- entity kinds are hardcoded from the reviewed list, not dynamically read.
