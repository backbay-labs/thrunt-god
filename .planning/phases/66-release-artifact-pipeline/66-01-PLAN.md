  <action>
Create shared release-contract utilities under `scripts/`:

1. Add `scripts/lib/obsidian-artifacts.cjs` exporting concrete constants and helpers:
   - `OBSIDIAN_INSTALL_ASSETS = ['main.js', 'manifest.json', 'styles.css']`
   - `OBSIDIAN_RELEASE_ASSETS = ['main.js', 'manifest.json', 'styles.css', 'versions.json']`
   - `getObsidianAppDir(repoRoot)`
   - `assertObsidianVersionSync({ rootPackage, obsidianPackage, manifest, versions })`
2. `assertObsidianVersionSync(...)` must throw when any of these drift:
   - root `package.json` version !== `apps/obsidian/package.json` version
   - `apps/obsidian/package.json` version !== `apps/obsidian/manifest.json` version
   - `apps/obsidian/versions.json[rootVersion]` is missing or does not equal `manifest.minAppVersion`
3. Add `scripts/build-obsidian-release.cjs` with concrete behavior:
   - repo root defaults to `path.join(__dirname, '..')`
   - reads root package + Obsidian package + manifest + versions via the shared helper
   - runs the existing root build entrypoint via `npm run build:obsidian`
   - creates `dist/obsidian-release/`
   - copies `main.js`, `manifest.json`, `styles.css`, and `versions.json` into that directory
   - prints a JSON payload describing the output directory and bundled asset filenames
4. The release bundle script must delete stale managed files in `dist/obsidian-release/` before rewriting them.
  </action>
  <verify>
    <automated>node scripts/build-obsidian-release.cjs</automated>
  </verify>

    - `scripts/build-obsidian-release.cjs` fails on version drift between root package, Obsidian package, manifest, and versions map
    - `scripts/build-obsidian-release.cjs` invokes the root `build:obsidian` workflow rather than bypassing it with a duplicate direct app build path
    - `node scripts/build-obsidian-release.cjs` creates `dist/obsidian-release/main.js`
    - `node scripts/build-obsidian-release.cjs` creates `dist/obsidian-release/manifest.json`
    - `node scripts/build-obsidian-release.cjs` creates `dist/obsidian-release/styles.css`
