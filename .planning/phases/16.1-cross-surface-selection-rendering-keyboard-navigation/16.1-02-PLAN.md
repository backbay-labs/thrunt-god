<!-- useRovingTabindex hook signature and usage pattern from Drain Template Viewer -->

From webview/shared/hooks/useRovingTabindex.ts:
```typescript
export function useRovingTabindex(
  containerRef: RefObject<HTMLElement>,
  itemSelector: string,
): void
```

Usage in webview/drain-template-viewer/app.tsx:
```typescript
const pinnedListRef = useRef<HTMLDivElement>(null);
useRovingTabindex(pinnedListRef, '.pinned-chip');
// ...
<div ref={pinnedListRef} role="list" aria-label="Pinned templates">
  {pinnedTemplates.map(t => (
    <button class="pinned-chip" role="listitem" ...>
  ))}
</div>
```

Key pattern:
1. Create a ref for the container element
2. Call useRovingTabindex(ref, CSS-selector-for-items)
3. Add role="list" to container, role="listitem" to items (or role="group" with aria-labels)
4. Items must be focusable (button/a) or have tabindex

**Evidence Board (index.tsx):**

Graph mode keyboard navigation:
1. In GraphView component, create `const nodeListRef = useRef<HTMLDivElement>(null);`
2. Call `useRovingTabindex(nodeListRef, '.hunt-eb-node');`
3. Wrap the SVG in a div with `ref={nodeListRef}` -- NOTE: SVG `<g>` elements are not natively focusable. Instead, add a `<foreignObject>` based approach is too complex. Better approach: add a visually-hidden HTML overlay list that mirrors graph nodes for keyboard navigation, OR add tabindex="0" to each SVG `<g>` node group.
4. **Chosen approach (simpler):** Add `tabindex={isDimmed ? -1 : 0}` to each `<g class="hunt-eb-node">` element. Add `role="img"` and `aria-label={node.label + ' (' + node.type + (node.verdict ? ', ' + node.verdict : '') + ')'}` to each `<g>`.

6. Create a ref for the SVG container div: `const graphContainerRef = useRef<HTMLDivElement>(null);`
7. Call `useRovingTabindex(graphContainerRef, '.hunt-eb-node[tabindex]');`
8. Add `ref={graphContainerRef}` to the `.hunt-eb-graph` div.
9. Add keydown handler on each `<g>`: on Enter, trigger the same action as click (`handleNodeClick`). This makes keyboard-focused nodes activatable.
