**store.ts:**
Fix TypeScript narrowing error at line 418. The issue is that inside a lambda, TypeScript does not carry forward the `hunt.state.status === 'loaded'` narrowing from the outer `if` block.

Current code (around line 417-419):
```typescript
} else if (hunt.state.status === 'loaded') {
  const phaseName = phases.find(p => p.number === hunt.state.data.phase)?.name ?? '';
  suggestedAction = `Continue Phase ${hunt.state.data.phase}${phaseName ? `: ${phaseName}` : ''}`;
}
```

Fix: Extract `hunt.state.data` to a local `const` BEFORE the lambda/conditional block that uses it:
```typescript
} else if (hunt.state.status === 'loaded') {
  const stateData = hunt.state.data;
  const phaseName = phases.find(p => p.number === stateData.phase)?.name ?? '';
  suggestedAction = `Continue Phase ${stateData.phase}${phaseName ? `: ${phaseName}` : ''}`;
}
```

This satisfies the TypeScript narrowing because `stateData` is a const captured at the point where status is known to be 'loaded'.
