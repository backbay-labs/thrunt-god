**In surfaces/packages/surfaces-state/src/projections.ts**, add these 3 exported functions AFTER the existing `deriveHypothesisStats()`:

1. `deriveRecommendedActions(artifacts: RawArtifacts): RecommendedAction[]` -- Analyzes case state and returns prioritized action list:
   - If any hypotheses have status 'Open' and zero linked evidence (check receiptSummary.relatedHypotheses and querySummary.relatedHypotheses for that hypothesis ID), emit action: `{ id: 'evidence-gap-N', label: '{count} hypotheses have no evidence', priority: 'high', category: 'evidence_gap' }`
   - If progress has a phase with status 'planned' that is the next sequential after the current running phase, emit: `{ id: 'phase-ready', label: 'Phase {N} ready for execution', priority: 'medium', category: 'phase_progress' }`
   - If any captured evidence has `reviewStatus === 'pending'` or empty relatedHypotheses, emit: `{ id: 'capture-debt-N', label: '{count} captured evidence items need linking', priority: 'medium', category: 'capture_debt' }`
   - If total hypotheses > 0 and deriveHypothesisStats shows open > supported + disproved, emit: `{ id: 'coverage-low', label: 'Most hypotheses still open ({open}/{total})', priority: 'low', category: 'hypothesis_coverage' }`
   - Return sorted by priority (high first, then medium, then low), max 5 items.

2. `mergeEvidenceTimeline(artifacts: RawArtifacts): EvidenceTimelineEntry[]` -- Merges queries, receipts, and evidence into a single chronological list:
   - Map each query to `{ id: q.queryId, type: 'query', vendorId: q.connectorId, timestamp: q.executedAt, summary: q.title || q.queryId, relatedHypotheses: q.relatedHypotheses }`
   - Map each receipt to `{ id: r.receiptId, type: 'receipt', vendorId: r.connectorId, timestamp: r.createdAt, summary: r.claim || r.receiptId, relatedHypotheses: r.relatedHypotheses }`
   - Map each evidence to `{ id: e.evidenceId, type: 'evidence', vendorId: e.vendorId, timestamp: e.capturedAt, summary: e.summary, relatedHypotheses: e.relatedHypotheses }`
   - Sort descending by timestamp (most recent first), take first 20.

3. `deriveAdapterStatuses(artifacts: RawArtifacts, detectedVendorId: string | null): AdapterStatus[]` -- Builds status list from certification data:
   - Collect unique vendorIds from certification array and from queries/receipts/evidence connectorIds.
   - For each vendorId: if it matches `detectedVendorId`, state = 'connected'. Else if certification exists and status includes 'certified', state = 'certified'. Else if certification exists but status is something else, state = 'uncertified'. Else state = 'disconnected'.
   - DisplayName: capitalize vendorId and replace hyphens with spaces. E.g. 'elastic' -> 'Elastic', 'crowdstrike' -> 'Crowdstrike'.

Then update `projectCaseViewModel()` to populate the 3 new CaseViewModel fields by calling these functions:
```typescript
recommendedActions: deriveRecommendedActions(artifacts),
evidenceTimeline: mergeEvidenceTimeline(artifacts),
adapterStatuses: deriveAdapterStatuses(artifacts, null),
```
Note: `detectedVendorId` is null at the projection level since it depends on browser-side state. The sidepanel will re-derive with the detected vendor at render time.
