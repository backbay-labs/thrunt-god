    - Test 9: All methods that call MCP return null/fallback on error, never throw
    - Test 10: callTool returns null when not connected (graceful degradation)
  </behavior>
  <action>
    1. Add to `types.ts`:
       ```typescript
       export type McpConnectionStatus = 'disabled' | 'disconnected' | 'connected' | 'error';

       export interface McpHealthResponse {
         status: string;        // "healthy" or "unhealthy"
         toolCount: number;
         serverVersion: string;
         error?: string;        // present when status is "unhealthy"
       }

       export interface McpToolResult {
         content: Array<{ type: string; text: string }>;
         isError?: boolean;
       }
       ```

    2. Create `mcp-client.ts` with:
       - `McpClient` interface:
         - `getStatus(): McpConnectionStatus`
         - `isConnected(): boolean`
         - `connect(): Promise<void>`
         - `disconnect(): void`
         - `checkHealth(): Promise<McpHealthResponse | null>`
         - `callTool(name: string, args: Record<string, unknown>): Promise<McpToolResult | null>`
       - `HttpMcpClient` class implementing `McpClient`:
         - Constructor takes `getSettings: () => { mcpServerUrl: string; mcpEnabled: boolean }` and a `requestFn` parameter (defaults to Obsidian's `requestUrl` but injectable for testing)
         - `requestFn` signature: `(opts: { url: string; method: string; body?: string; headers?: Record<string, string> }) => Promise<{ status: number; text: string }>`
         - Private `status: McpConnectionStatus = 'disconnected'`
         - `getStatus()`: returns `'disabled'` if `!getSettings().mcpEnabled`, else returns internal `status`
         - `isConnected()`: returns `status === 'connected'`
         - `connect()`: calls `GET {serverUrl}/health`, parses JSON response, sets status to 'connected' if response.status === 'healthy', 'error' otherwise. On fetch error, sets status to 'error'. Never throws.
         - `disconnect()`: sets status to 'disconnected'
         - `checkHealth()`: if not connected returns null; otherwise calls health endpoint and returns parsed `McpHealthResponse`. On error returns null.
         - `callTool(name, args)`: if not connected returns null; calls `POST {serverUrl}/tool` with JSON body `{ name, args }`, returns parsed `McpToolResult`. On error returns null. Never throws.
       - `StubMcpClient` class implementing `McpClient` for testing:
         - Configurable responses via `setHealthResponse()` and `setToolResponse()`
         - Tracks call history for assertions

    3. Create `__tests__/mcp-client.test.ts`:
       - Use the behavior tests listed above
       - Test HttpMcpClient with an injected `requestFn` stub (no real HTTP calls)
       - Verify graceful degradation: every error path returns null, never throws

       new Setting(containerEl)
         .setName('Test connection')
         .setDesc('Verify the MCP server is reachable.')
         .addButton((button) =>
           button
