Create `tests/drain.test.cjs` with comprehensive unit tests for the DrainParser class. Use `node:test` with `describe`/`it` blocks following the existing test patterns in the codebase.

The test file requires `../thrunt-god/bin/lib/drain.cjs` directly (not through runtime.cjs).

**describe('DrainParser tokenization'):**
- Test that whitespace-separated content produces correct token count
- Test that extraDelimiters option splits tokens further (e.g., extraDelimiters: ['='] splits "key=value" into ["key", "value"])
- Test that empty/whitespace-only content returns null from addMessage

**describe('DrainParser pre-masking'):**
- Test IPv4 replacement: "Failed login from 192.168.1.50" masks IP to `<IP>`
- Test IPv6 replacement: content with "2001:0db8:85a3:0000:0000:8a2e:0370:7334" becomes `<IP>`
- Test UUID replacement: "session abc12345-de67-890f-abcd-ef1234567890" masks to `<UUID>`
- Test SHA-256 hash replacement (64 hex chars) masks to `<HASH>`
- Test SHA-1 hash replacement (40 hex chars) masks to `<HASH>`
- Test MD5 hash replacement (32 hex chars) masks to `<HASH>`
- Test ISO timestamp replacement masks to `<TS>`
- Test syslog timestamp replacement masks to `<TS>`
- Test email replacement masks to `<EMAIL>`
- Test Windows path replacement masks to `<WINPATH>`
- Test Unix path replacement masks to `<PATH>`
- Test that masking produces stable clustering: two messages differing only in IP/UUID cluster together
- Test custom maskPatterns option overrides DEFAULT_SECURITY_MASKS
- Test empty maskPatterns=[] disables masking

**describe('DrainParser tree search and clustering'):**
- Test that identical messages produce same cluster (changeType: 'none' after first)
- Test that "Failed password for admin from <IP>" and "Failed password for root from <IP>" cluster together after masking
- Test that messages with different token counts go to different clusters
- Test parametrizeNumericTokens=true routes "port 8080" and "port 443" to wildcard (same cluster)
- Test parametrizeNumericTokens=false keeps numeric tokens literal (separate clusters)
- Test that simTh controls cluster granularity: low threshold merges more, high threshold keeps separate

**describe('DrainParser template merging'):**
- Test that differing tokens are replaced with paramStr '<*>'
- Test that changeType is 'cluster_template_changed' when template merges
- Test that cluster ID changes after template merge (content-hash-based)
- Test that merged template preserves common tokens and wildcards differing positions

**describe('DrainParser match mode'):**
- Test match() returns cluster info for known content
- Test match() returns null for unknown content
- Test match() does NOT increment totalMessages (getStats check before/after)
- Test match() does NOT change cluster size (getClusters check before/after)
- Test match() does NOT modify template (addMessage, match new variant, template unchanged)

**describe('DrainParser serialization'):**
- Test toJSON() returns version:1 with config, totalMessages, clusters
- Test fromJSON(toJSON()) produces parser that matches same content identically
- Test fromJSON rejects state without version 1 (throws Error)
- Test fromJSON with maskPatterns option override restores masking behavior
- Test round-trip preserves cluster count and template text
- Test getStats() on restored parser matches original

**describe('DrainParser introspection'):**
- Test getClusters() returns Map with correct template and size
- Test getCluster(id) returns info for valid ID, null for invalid
- Test getStats() returns clusterCount and totalMessages
- Test getClusters() returns copies (mutation does not affect parser)

**describe('DrainParser configuration'):**
- Test depth minimum is 3 (passing depth=1 results in depth=3)
- Test maxClusters cap: after reaching limit, addMessage returns null for new clusters
- Test maxChildren limits tree branching
- Test custom paramStr replaces default '<*>'

**describe('DrainParser content-hash IDs'):**
- Test same template text always produces same cluster ID across parser instances
- Test different template text produces different cluster IDs
- Test cluster ID is 16 hex characters

**describe('DEFAULT_SECURITY_MASKS'):**
- Test that DEFAULT_SECURITY_MASKS is a frozen array (Object.isFrozen)
- Test that it contains 13 mask patterns
- Test patterns are ordered: SHA-256 before SHA-1 before MD5 (most-specific first)
