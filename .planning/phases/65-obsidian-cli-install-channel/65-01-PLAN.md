Add deterministic Obsidian bundle helpers inside `bin/install.js`:

1. Require `node:child_process` via `const { execFileSync } = require('node:child_process');`.
2. Define shared constants near the installer helpers:
   - `const OBSIDIAN_PLUGIN_ID = 'thrunt-god';`
   - `const OBSIDIAN_ASSET_FILES = ['main.js', 'manifest.json', 'styles.css'];`
3. Add helper `getObsidianStageDir(homeDir = os.homedir())` returning `path.join(homeDir, '.thrunt', 'obsidian')`.
4. Add helper `buildObsidianBundle(repoRoot, runBuild = execFileSync)` that executes:
   - `npm run build:obsidian` with `cwd: repoRoot`
   - `stdio: 'inherit'`
   - throws if the command fails
5. Add helper `stageObsidianBundle(options)` with concrete behavior:
   - defaults: `repoRoot = path.join(__dirname, '..')`, `stageDir = getObsidianStageDir()`, `pluginDir = path.join(repoRoot, 'apps', 'obsidian')`
   - ensures `stageDir` exists
   - copies `main.js`, `manifest.json`, and `styles.css` from `apps/obsidian/` into `stageDir`
   - removes stale copies of those three managed files before rewriting them
   - verifies each staged file exists after copying and throws if any are missing
6. Add `installObsidian()` entrypoint shell:
   - logs `Installing THRUNT for Obsidian to ~/.thrunt/obsidian`
   - calls `buildObsidianBundle(...)`
   - calls `stageObsidianBundle(...)`
   - returns the staged directory path and staged file list for later vault-link work in Plan 02
7. Under `THRUNT_TEST_MODE`, export the new helpers so tests can call them directly:
   - `getObsidianStageDir`
   - `buildObsidianBundle`
   - `stageObsidianBundle`
   - `OBSIDIAN_PLUGIN_ID`
   - `OBSIDIAN_ASSET_FILES`
  </action>
