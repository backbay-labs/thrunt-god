# Receipt: david.park Account Compromised via MFA Push Acceptance

## Claim

david.park@meridian.io was compromised via credential stuffing (valid password from breach database) with MFA bypass through a single push notification acceptance. The attacker then established persistence by swapping MFA factors and changing the account password (HYP-02).

## Evidence

**events=28, templates=4, entities=1** (from QRY-20260329-002)

Observable facts from the david.park entity timeline:

1. **Credential compromise:** Successful authentication from 198.51.100.42 at 14:10:33Z, exactly 1 minute 46 seconds after the spray ended (14:08:47Z). Source IP 198.51.100.42 resolves to AS396982, the same residential proxy ASN used in the spray.

2. **MFA acceptance:** Single Okta Verify push notification sent at 14:10:58Z, accepted at 14:11:02Z (4-second response time). No prior push denials. The single-push acceptance with no denials suggests either: (a) MFA fatigue from a social engineering call ("IT is testing your MFA, please approve"), or (b) user habitually approves pushes without reading. No evidence of phone call in Okta logs (blind spot).

3. **User-agent anomaly:** Attacker session used `Mozilla/5.0 (Windows NT 10.0; Win64; x64)`. david.park's baseline is `Mozilla/5.0 (Macintosh; Intel Mac OS X 14_2_1)`. david.park does not have a Windows device in Okta device inventory.

4. **MFA factor swap (persistence):**
   - 14:31:10Z: New SMS factor enrolled (phone: +1-XXX-XXX-7734)
   - 14:31:42Z: New SMS factor activated
   - 14:32:15Z: Original Okta Verify push factor deactivated
   - Net effect: attacker replaced david.park's MFA with attacker-controlled phone number

5. **Recovery information changed:**
   - 14:33:00Z: Recovery phone number changed
   - 14:33:30Z: Recovery email changed
   - These changes ensure the legitimate user cannot self-service recover the account

6. **Password changed:**
   - 14:56:00Z: Password changed via user self-service (from the attacker's session)
   - 14:58:00Z: Re-authentication with new password confirmed
   - This completes the account takeover -- original credentials no longer work
